# Experiments

13 training runs recovered from the HPC experiment directories
(`experiments/<name>/metrics/{summary.json,train_history.jsonl}`) and Slurm
`sacct`. Raw logs are committed under
[`assets/experiment_logs/`](assets/experiment_logs/); figures are regenerated by
`scripts/make_portfolio_figures.py`.

## Naming convention

`<identity>_<model>[_<variant>]-<YYYYMMDD-HHMMSS>`, e.g. `huberman_flowcfg16-20260710-161733`
= Huberman identity, flow+CFG model, 16-clip dataset, started 2026-07-10 16:17:33.
Each run writes `metrics/summary.json`, `metrics/train_history.jsonl`,
`config.resolved.json`, and `checkpoints/{best,last}.pt`.

## Sync-confidence (cross-model, SyncNet-scored)

Measured with `scripts/eval_sync_confidence.py` on `huberman_session0_000`
(ground truth = 0.77):

| Model | Sync-confidence | Offset peak |
|---|---|---|
| MSE GRU (huberman_gru_w2v) | 0.09 | −7 (misaligned) |
| Flow (8-clip) | 0.43 | −10 |
| Flow + CFG 16-clip, w=2.0 | 0.54 | **0** (aligned) |
| Flow + CFG 16-clip, w=3.0 | 0.56 | **0** |

## Progression (what each step bought)

1. **GRU + prosody → GRU + wav2vec2:** richer audio features lower motion-regression loss.
2. **GRU (MSE) → flow matching:** the decisive jump — MSE mean-collapses to a
   near-static face (sync 0.09); generative sampling restores motion (0.43).
3. **+ CFG:** guidance weight `w≥2` tightens lip-sync (0.43 → 0.54) and fixes
   time alignment (offset −10 → 0).
4. **+ 2× data (8→16 clips):** better generalization (val loss 0.29 → 0.23, sync → 0.56).

## Ablations available from existing runs

- **Audio encoder:** prosody (5-d) vs log-mel (42-d) vs wav2vec2 (768-d) — GRU runs above.
- **Directionality:** `gru_mel` vs `gru_melbi` (bidirectional).
- **Regularization:** `gru_w2v` vs `gru_w2vreg`.
- **Objective:** GRU-MSE vs flow-matching (sync-confidence table).
- **Guidance weight:** w ∈ {1, 1.5, 2, 3} (CFG sweep figure).
- **Dataset size:** 8-clip vs 16-clip flow+CFG.
